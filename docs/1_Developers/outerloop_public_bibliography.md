# Public outer-loop reconstruction bibliography

This implementation uses only repository artifacts and public literature. Capability evidence is separated from independently designed details when a source does not disclose a reproducible algorithm.

1. Englander, Conway, and Williams, “Automated Mission Planning via Evolutionary Algorithms,” *Journal of Guidance, Control, and Dynamics* 35(6), 2012, [doi:10.2514/1.54101](https://doi.org/10.2514/1.54101). Public basis for the nested discrete evolutionary outer loop and continuous EMTG inner loop.
2. Englander, “Automated trajectory planning for multiple-flyby interplanetary missions,” UIUC dissertation, 2013, [IDEALS 2142/44367](https://hdl.handle.net/2142/44367). Public basis for null-gene transcription, automatic sequence selection, and MBH/NLP nesting; the repository copy is metadata-only/withdrawn, so unavailable details are not inferred.
3. Englander and Vavrina, “Multiobjective Hybrid Optimal Control for Multiple-Flyby Interplanetary Mission Design,” AAS 15-523, 2015, [NASA NTRS 20150020820](https://ntrs.nasa.gov/citations/20150020820). Documents NSGA-II, cap-and-optimize menus, feasibility-first selection, MPI isolation, and memoization.
4. Vavrina, Englander, and Ellison, “Global Optimization of Multiple-Flyby Low-Thrust Missions Using the Evolutionary Mission Trajectory Generator,” AAS 16-272, 2016, [NASA NTRS 20160001644](https://ntrs.nasa.gov/citations/20160001644). Basis for variable flyby sequences and MGAnDSMs architectural selection.
5. Englander et al., multi-vehicle design with null genes, 2018, [NASA NTRS 20180006444](https://ntrs.nasa.gov/citations/20180006444). Confirms fixed maximum arrays with inactive insertion/deletion semantics.
6. Englander et al., EMTG overview, 2019, [NASA NTRS 20200001624](https://ntrs.nasa.gov/citations/20200001624). Summarizes outer-loop destination/flyby/hardware choices and PEATSA sharing.
7. Hughes, Knittel, and Englander, “PEATSA,” 2017, [NASA NTRS 20170011199](https://ntrs.nasa.gov/citations/20170011199), and the Lunar IceCube application, [NASA NTRS 20205004857](https://ntrs.nasa.gov/citations/20205004857). Basis for iterative neighboring-case seed transfer and fidelity progression.
8. Deb et al., “A Fast and Elitist Multiobjective Genetic Algorithm: NSGA-II,” 2002, [doi:10.1109/4235.996017](https://doi.org/10.1109/4235.996017). Algorithmic baseline.
9. Abdelkhalik and Darani, “Hidden Genes Genetic Algorithms for Systems Architecture Optimization,” GECCO 2016, [doi:10.1145/2908812.2908949](https://doi.org/10.1145/2908812.2908949). Basis for retained payload plus binary visibility tags.
10. Ellithy et al., “Mutation Operator for Resonance Flybys in Moon Tour Design Optimization,” 2024, [doi:10.2514/1.G007708](https://doi.org/10.2514/1.G007708). The accessible record establishes a resonance mutation integrated with HGGA/EMTG, but not enough detail for exact reconstruction. The included ratio/turning screen is independently designed and labeled accordingly.
11. Vijayakumar and Abdelkhalik, “An Initial Guess Tool for EMTG,” AIAA 2022-2391, [doi:10.2514/6.2022-2391](https://doi.org/10.2514/6.2022-2391). Basis for the external R-FFS provider boundary and staged TrialX use; no claim is made that the core implements R-FFS.

Repository forensic inputs include `MissionOptionsV1.py`, `JourneyOptionsV1.py`, the historical NSGAII reader/panels/filter, `bulletproof.py`, PEATSA seed/oven/menu code, current generated options APIs, decision-vector converters, `problem.cpp`, executable output behavior, universe/hardware formats, mission factories, Testatron, tutorials, and examples. V1 and PEATSA modules are evidence and compatibility inputs, not production foundations.
